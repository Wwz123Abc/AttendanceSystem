# 考勤系统 新服务器首次部署指南（Linux）

版本：v1.0　日期：2026-07-14

## 写在前面：这份文档解决什么问题

项目里已有 `docs/部署手册.docx` 和 `docs/环境迁移配置清单.docx`，但那两份文档默认"服务已经在跑了"，讲的是"怎么升级"和"迁移时要改哪些配置"，**没有讲"一台全新的空服务器，怎么从零跑起来这个系统"**——这正是交接给公司 IT、用一台新服务器时最先要解决的问题。这份文档补这一段。

以下步骤基于 **Linux（以 Ubuntu 22.04 为例，其他发行版命令细节略有差别）**。如果公司实际给的是 Windows Server，这套方案不适用，需要另外准备一份 IIS/Windows 服务版的部署指南——请先确认好服务器系统再照做。

## 〇、先决定一件事：这台服务器要不要能被公网访问

这决定了后面要不要做域名/HTTPS证书/ICP备案这几步，两条路线复杂度差很多，先想清楚：

| 场景 | 要不要域名+HTTPS+备案 |
|---|---|
| 只在公司内网/WiFi 里用电脑或手机浏览器访问，不需要员工在家/外面打卡 | **不需要**，直接用服务器内网 IP + 端口访问即可，简单很多 |
| 需要员工在公司网络之外也能访问（比如出差打卡、在家审批） | 需要域名 + HTTPS证书 + （如果是中国大陆服务器且要用 80/443 端口）ICP 备案 |

本系统对钉钉的调用都是"服务器主动去查钉钉"（导入员工、同步打卡记录），钉钉不会反过来调用本系统，所以**不需要公网入口钉钉也能正常同步**，只有"员工要不要在公司网络之外访问系统本身"这一点决定要不要公网/域名/证书。

下面步骤按"两种都要考虑到"来写，域名/HTTPS 相关的步骤标了「仅公网访问需要」，纯内网部署可以跳过。

## 一、服务器准备工作

### 1.1 最低配置建议
- 2 核 2G 内存以上（并发人数多、月度报表人数多可以再加）
- 系统盘预留至少 10GB（程序 + 数据库 + 日志 + 员工上传的照片/附件）
- Ubuntu 22.04 LTS（或公司标准 Linux 发行版）

### 1.2 需要开放的端口
- 22（SSH，仅管理员/IT 能访问，不要对公网全开）
- 80、443 — **仅公网访问需要**
- 内网访问不需要额外开放端口给"其他人"，但要保证公司内网设备能访问服务器所在网段

### 1.3 安装 .NET 8 运行时
```bash
# Ubuntu
wget https://packages.microsoft.com/config/ubuntu/22.04/packages-microsoft-prod.deb -O packages-microsoft-prod.deb
sudo dpkg -i packages-microsoft-prod.deb
sudo apt update
sudo apt install -y aspnetcore-runtime-8.0
dotnet --list-runtimes   # 确认看到 Microsoft.AspNetCore.App 8.0.x
```

### 1.4 安装数据库（MariaDB，兼容 MySQL 协议）
```bash
sudo apt install -y mariadb-server
sudo mysql_secure_installation   # 按提示设置 root 密码等安全选项

sudo mysql -uroot -p
```
进入 MySQL 命令行后建库建账号（把 `换一个强密码` 换成实际密码，记下来，后面配置连接串要用）：
```sql
CREATE DATABASE AttendanceDB CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;
CREATE USER 'att_user'@'localhost' IDENTIFIED BY '换一个强密码';
GRANT ALL PRIVILEGES ON AttendanceDB.* TO 'att_user'@'localhost';
FLUSH PRIVILEGES;
EXIT;
```
⚠️ 数据库端口（默认 3306）**不要对公网开放**，只允许本机（`localhost`）连接，这也是上面用 `'att_user'@'localhost'` 而不是 `'%'` 的原因。

### 1.5（仅公网访问需要）安装 Nginx + certbot
```bash
sudo apt install -y nginx certbot python3-certbot-nginx
```

## 二、本地：发布程序

在开发者本机（现在这台电脑）执行：
```powershell
cd "C:\Users\w\Desktop\AttendanceSystem(1)\AttendanceSystem"
dotnet publish -c Release -o .\publish
tar -czf publish.tar.gz -C .\publish .

# 首次部署，数据库是空的，需要生成完整的建表脚本（不是"增量迁移"）：
dotnet ef migrations script -o migrate.sql
```
把 `publish.tar.gz` 和 `migrate.sql` 传到新服务器（用 `scp`、或者 IT 习惯的其它文件传输方式都行）：
```powershell
scp publish.tar.gz migrate.sql root@新服务器IP:/root/
```

## 三、服务器：部署程序

### 3.1 解压到程序目录
```bash
sudo mkdir -p /var/www/attendance-net
sudo tar -xzf /root/publish.tar.gz -C /var/www/attendance-net
sudo useradd -r -s /usr/sbin/nologin www-data 2>/dev/null   # 大多数发行版已自带 www-data，报错可忽略
sudo chown -R www-data:www-data /var/www/attendance-net
```

### 3.2 建表（首次，数据库是空的）
```bash
mysql -uatt_user -p AttendanceDB < /root/migrate.sql
```

### 3.3 配置生产环境连接串等敏感信息
生产环境真正生效的是 `appsettings.Production.json`（优先级比代码包里的 `appsettings.json` 高，且不用打进发布包，改这里最安全）：
```bash
sudo nano /var/www/attendance-net/appsettings.Production.json
```
内容（按实际值替换）：
```json
{
  "ConnectionStrings": {
    "Default": "Server=localhost;Port=3306;Database=AttendanceDB;User=att_user;Password=换成刚才建库时设的密码;CharSet=utf8mb4;AllowPublicKeyRetrieval=true;SslMode=None;"
  },
  "DingTalk": {
    "AppKey": "换成公司真实钉钉应用的 AppKey",
    "AppSecret": "换成公司真实钉钉应用的 AppSecret",
    "AgentId": "换成公司真实钉钉应用的 AgentId"
  }
}
```
钉钉应用的 AppKey/AppSecret/AgentId 在钉钉开放平台的自建应用详情页能看到；如果这台服务器要用于同步钉钉打卡数据，还需要：
- 在钉钉开放平台，把这个自建应用的可用范围设为"全部员工"（或至少覆盖需要考勤的人）；
- 把服务器的**公网出口 IP** 加入应用的 IP 白名单（否则调用钉钉接口会报权限错误）——注意是服务器访问外网时对外显示的 IP，不是内网 IP。

### 3.4 配置成 systemd 服务（让它开机自启、崩了自动重启）
```bash
sudo nano /etc/systemd/system/attendance-net.service
```
内容：
```ini
[Unit]
Description=考勤系统 (AttendanceSystem)
After=network.target mysql.service

[Service]
WorkingDirectory=/var/www/attendance-net
ExecStart=/usr/bin/dotnet /var/www/attendance-net/AttendanceSystem.dll
Restart=always
RestartSec=5
KillSignal=SIGINT
SyslogIdentifier=attendance-net
User=www-data
Environment=ASPNETCORE_ENVIRONMENT=Production
Environment=ASPNETCORE_URLS=http://127.0.0.1:5080

[Install]
WantedBy=multi-user.target
```
说明：程序监听 `127.0.0.1:5080`，**只在服务器本机可访问**，不直接对外/对内网暴露——公网访问走 Nginx 反向代理转发（见下一步）；纯内网访问则直接把 `127.0.0.1` 换成 `0.0.0.0`（监听所有网卡），让内网设备能直接用 `服务器内网IP:5080` 访问，不需要 Nginx。

启动服务：
```bash
sudo systemctl daemon-reload
sudo systemctl enable attendance-net
sudo systemctl start attendance-net
sudo systemctl status attendance-net --no-pager   # 期望看到 active (running)
```

### 3.5（仅公网访问需要）配置 Nginx 反向代理 + HTTPS
```bash
sudo nano /etc/nginx/sites-available/attendance-net
```
内容（把 `你的域名` 换成实际域名）：
```nginx
server {
    listen 80;
    server_name 你的域名;

    location / {
        proxy_pass         http://127.0.0.1:5080;
        proxy_set_header   Host $host;
        proxy_set_header   X-Real-IP $remote_addr;
        proxy_set_header   X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header   X-Forwarded-Proto $scheme;
    }

    client_max_body_size 20m;   # 员工上传身份证照片/审批附件，默认限制太小会传不上
}
```
```bash
sudo ln -s /etc/nginx/sites-available/attendance-net /etc/nginx/sites-enabled/
sudo nginx -t   # 测试配置没写错
sudo systemctl reload nginx

# 签发 HTTPS 证书（会自动改写上面的 nginx 配置加上 443/证书路径）
sudo certbot --nginx -d 你的域名
```
在此之前，域名服务商那边要先把这个域名的 A 记录指向服务器公网 IP，且（中国大陆服务器）域名要完成 ICP 备案，否则 80/443 会被运营商拦截，证书签发和访问都会失败。

## 四、验证

```bash
# 服务器本机
curl -I http://127.0.0.1:5080/Login   # 期望 200 或 302
systemctl status attendance-net --no-pager

# 浏览器
# 内网：http://服务器内网IP:5080
# 公网：https://你的域名
```
打开后应该能看到登录页；首次启动、数据库刚建好表时，系统会自动创建默认管理员账号（工号 `admin`，密码见 `docs/环境迁移配置清单.docx` 第四节），**登录后立即改密码**。然后到"员工管理"页试一下"从钉钉导入员工"，能成功或至少给出明确的错误提示（不是白屏/500），说明钉钉这块也接通了。

## 五、部署完成后

- 以后的日常更新（改了代码/加了字段），走 `docs/部署手册.docx` 里的"标准部署"/"带数据库变更的部署"流程即可，不用再重复本文档的步骤。
- `deploy.ps1` 脚本顶部的 `$Server` 参数要改成新服务器的地址；服务器上暂时还没有 `upgrade.sh`（这是老服务器上手工维护的脚本，本项目代码库里没有备份），首次部署完先只用"手动分步版"更新几次，等确认流程稳定后再考虑照着 `docs/部署手册.docx` 里描述的行为补一份 `upgrade.sh` 到新服务器上，方便以后一键升级。
- 提醒 IT：`wwz.cool` 现在的服务器如果之后要下线，记得先把 `wwz.cool` 域名解析和证书处理掉，避免旧服务器关闭后域名还指向一个死地址。
