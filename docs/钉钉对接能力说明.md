# 钉钉对接能力说明

> 这份文档说明"考勤系统"目前和钉钉对接了哪些功能：能从钉钉拿到什么数据、能通过系统改钉钉那边的什么东西，
> 以及钉钉官方开放平台本身还能做哪些我们目前没用上的事情，方便以后评估要不要继续接入更多能力。
> 语言尽量说人话，技术细节放在括号里，可以跳过不看。

---

## 一句话总结

**目前的对接绝大部分是"只读"的**：从钉钉往系统里搬数据（通讯录、打卡、请假）。
系统这边真正会**反过来改动钉钉那边数据**的，只有两个：

1. **发工作通知**（给员工手机上钉钉推一条消息，比如找回密码的验证码）——这个不算"改数据"，只是发消息。
2. **删除员工**（管理员在系统里彻底删除一个绑定过钉钉的员工时，会顺带把这个人从钉钉通讯录里也删掉）——这是唯一一个真正会**改动钉钉后台数据**的操作。

除此之外，导入部门、导入员工、拉打卡记录、拉请假记录，全部都是"钉钉 → 本系统"单向的，不会对钉钉那边的任何数据做增删改。

---

## 一、需要的钉钉配置

对接需要在钉钉开发者后台建一个"企业内部应用"，拿到下面这三样东西，填进系统的配置文件（`appsettings.json` 的 `DingTalk` 节）：

| 配置项 | 用途 |
|---|---|
| AppKey / AppSecret | 应用身份凭证，换取访问令牌（access_token）用 |
| AgentId | 发"工作通知"专用，和上面两个不是一回事 |
| EnableLeaveSync | 是否连请假数据也一起同步（需要额外开通"考勤/假期数据"权限） |
| EnableScheduledSync | 是否开启后台自动定时同步打卡（默认关闭，需要管理员手动点"立即同步"） |

也就是说，这个应用需要在钉钉后台至少开通："通讯录只读权限"、"考勤数据权限"、"工作通知权限"、"用户管理（删除员工）权限"这四类接口权限，钉钉才会放行对应的调用。

## 二、从钉钉读取的数据（钉钉 → 本系统）

### 1. 通讯录导入（部门 + 员工）
- **触发方式**：员工管理页的"从钉钉导入员工"按钮
- **拿到什么**：整个企业的部门结构（部门名称、上下级关系）、每个员工的姓名、手机号、工号、职位、入职日期、钉钉账号 ID
- **落地效果**：本系统里没有的部门/员工会新建，已有的员工会按"钉钉账号 ID"或"工号"自动对上、补全信息，不会覆盖管理员在系统里已经手动填过的部门/考勤组等字段

### 2. 按工号自动关联账号
- **触发方式**："自动映射 userid" 按钮
- **拿到什么**：钉钉通讯录里每个人的"工号 → 钉钉账号 ID"对应关系
- **落地效果**：系统里已经存在、但还没关联钉钉账号的员工，按工号自动对上，之后就能收打卡数据和工作通知了

### 3. 打卡记录同步
- **触发方式**："立即同步打卡"按钮（手动点，选日期范围）；也可以在配置里打开"后台自动定时同步"，不用人管
- **拿到什么**：员工在钉钉 APP 上的打卡时间、打卡地点（经纬度+地址）、迟到/早退/旷工等钉钉自己的判定结果
- **落地效果**：写成本系统的打卡流水和每日考勤记录，并且会拿打卡定位和系统里配置的考勤组打卡地点做二次比对，对不上会标"定位异常"提醒管理员核实（这是系统自己做的校验，不是钉钉给的）

### 4. 请假记录同步（可选功能）
- **触发方式**：跟打卡同步一起自动执行（需要打开 EnableLeaveSync 配置，且钉钉那边要开通"考勤/假期数据"权限）
- **拿到什么**：员工在钉钉上请假审批通过的时间段、假期类型
- **落地效果**：把请假区间内每天的考勤状态改成"请假"；如果没开通权限，这部分会自动跳过，不影响打卡同步

## 三、通过系统修改钉钉那边数据（本系统 → 钉钉）

### 1. 发工作通知（不算"改数据"，是发消息）
- **触发场景**：员工在登录页点"忘记密码"，系统会把 6 位验证码通过钉钉工作通知发到员工本人手机上
- **说明**：这个操作只是给指定的人发一条消息，不会改动钉钉通讯录/考勤等任何数据

### 2. 删除员工
- **触发场景**：管理员在"员工管理"页点"彻底删除"某个绑定过钉钉的员工
- **实际效果**：系统会调用钉钉的"删除用户"接口，把这个人从钉钉通讯录里也删掉。根据钉钉官方说明，这个操作还会**连带把该员工从钉钉那边的考勤组、智能人事花名册里也删掉**（钉钉那边会把这个人标记为"已离职"）
- **风险提醒**：这是**不可逆**操作——以后如果要恢复，对方需要重新扫码/接受邀请才能重新加入钉钉通讯录，系统这边没法用接口把人自动加回去。如果钉钉那边删除失败（网络问题等），系统会照常完成本地删除，只是提示"请到钉钉通讯录手动确认"，不会因此卡住

### 3. 更新员工资料
- **触发场景**：管理员在"员工管理"页编辑一个**已经绑定过钉钉**（有 DingTalkUserId）的员工，改了姓名/手机号/工号/职位
- **实际效果**：保存本地信息的同时，调用钉钉的"更新用户"接口，把这几项同步改到钉钉通讯录里对应的那个人身上
- **不做的事**：一个纯本地新建、还没绑定钉钉的员工（比如很多临时工），改资料不会触发钉钉调用
- 钉钉那边更新失败（网络问题等）不会拦住本地保存，只会提示"请到钉钉通讯录手动确认"，具体错误原因会直接显示在提示里
- **已知限制（errcode=40022）**：如果员工本人登录钉钉时用的手机号，跟系统里存的手机号不一样，钉钉会拒绝这次更新（钉钉官方说法是"可以删除后重新添加"才能改，本系统没做这一步）。系统遇到这种情况会自动把手机号从这次同步里去掉，改成只同步姓名/工号/职位，不会因为手机号这一项卡住其余资料

### 4. 新建员工
- **触发场景**：管理员在"员工管理"页新增一名员工，且**同时满足两个条件**：填了手机号（钉钉建号必填、企业内必须唯一）、所选部门是从"从钉钉导入通讯录"关联过来的（能换算出对应的钉钉部门编号）
- **实际效果**：本地保存成功后，顺带调用钉钉的"创建用户"接口在钉钉通讯录里也建一个新账号，并把钉钉分配的 userid 存回本地这条员工记录，以后编辑/删除都会继续联动
- **不做的事**：两个条件缺一个（没填手机号，或部门是本地手工建的、换算不出钉钉部门编号）就直接跳过，不会强行创建，也不算错误——这是本系统很多临时工的常见情况
- **风险提醒**：钉钉建账号这个动作本身**不可逆**（钉钉那边只能"删除"不能"撤销"新建），如果手机号在企业内已被别人占用会直接创建失败；失败不会拦住本地新建，只会提示"请到钉钉通讯录手动添加"

## 四、钉钉官方还能做、但本系统目前没用上的能力

以下是钉钉开放平台官方文档里存在、但本系统代码里完全没有调用的能力，供以后参考：

| 能力 | 官方是否支持 | 本系统现状 |
|---|---|---|
| 创建/更新/删除部门（把系统里新建的部门同步到钉钉） | 支持 | 未实现，本系统的部门管理和钉钉部门是分开的（只在"导入"时单向读一次） |
| 打卡补卡（通过接口反向修正钉钉那边的打卡记录） | 支持（考勤写操作权限，需要单独申请） | 未实现，本系统的"补卡审批"只改自己数据库，不会回写钉钉 |
| 考勤组/班次查询与配置 | 支持（`获取用户考勤组`等接口） | 未实现，本系统的考勤组/班次是完全独立的一套，不读钉钉那边的排班 |
| 审批流程对接（把系统里的请假/加班申请提交到钉钉审批走一遍） | 支持（OA 审批开放接口） | 未实现，本系统的审批流程是自己独立的一套（员工在系统里申请、系统里的人审批），跟钉钉审批没有关系 |
| 企业考勤统计报表接口 | 支持 | 未实现，本系统自己算月度统计，不依赖钉钉的统计接口 |
| 群消息/机器人/日历/会议等其他钉钉能力 | 支持 | 未涉及，这套系统只用了钉钉的通讯录、考勤、工作通知三块 |

简单说：**这套系统把钉钉当成"数据来源"和"通知通道"来用，钉钉的组织架构、考勤规则本身并不受本系统管理**——员工的部门/考勤组/班次这些"管理规则"，都是在本系统里独立配置的，只是员工身份信息和原始打卡数据从钉钉搬过来。

## 五、涉及的代码文件（供以后维护参考）

| 文件 | 作用 |
|---|---|
| `Models/Options/DingTalkOptions.cs` | 钉钉相关配置项 |
| `Services/Implementations/DingTalkTokenProvider.cs` | 获取并缓存访问令牌 |
| `Services/Implementations/DingTalkContactClient.cs` | 拉通讯录（部门+员工）、删除员工 |
| `Services/Implementations/DingTalkAttendanceClient.cs` | 拉打卡结果 |
| `Services/Implementations/DingTalkLeaveClient.cs` | 拉请假状态 |
| `Services/Implementations/DingTalkNotifyClient.cs` | 发工作通知（验证码等） |
| `Services/Implementations/DingTalkSyncService.cs` | 把上面几个客户端串起来：导入员工、自动映射、同步打卡+请假 |
| `Services/BackgroundServices/AttendanceBackgroundService.cs` | 可选的后台自动定时同步（默认关闭） |
| `Services/Implementations/PasswordResetService.cs` | 忘记密码流程，用到工作通知发验证码 |
| `Services/Implementations/UserService.cs`（`DeleteUserAsync`） | 彻底删除员工时联动调用钉钉删除接口 |
| `Pages/Admin/UserManage.cshtml`（钉钉对接卡片） | 管理员操作入口：导入员工、自动映射、立即同步打卡 |
