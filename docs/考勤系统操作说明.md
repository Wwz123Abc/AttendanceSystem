## 一、登录与角色

### 1.1 登录页面

- 浏览器打开系统地址，输入工号和密码，点"登录"。
- 勾选"记住我"可在 8 小时内免登录。
- 忘记密码：登录页有"忘记密码？"链接，**新功能**——不用再找管理员，自己在页面上用"工号 + 手机号"验证身份，系统会通过钉钉给你绑定的账号发一条工作通知，里面有 6 位验证码，填上就能自己改密码（前提是你的账号已经绑定过钉钉，且手机号和账号里登记的一致）。
- 首次使用：管理员会为你创建账号并告知初始密码，登录后请立即在"改密码"里改掉。

### 1.2 角色与权限

系统分 **5 种角色**，登录后看到的菜单不同：

| 角色 | 权限范围 |
|---|---|
| 管理员 | 全部功能：员工、部门、考勤组、班次、假期、报表、审批处理 |
| 文员 | 和管理员权限范围基本一致（后台管理 + 报表 + 审批处理），差别在角色定位不是"正式工" |
| 主管 | 审批处理（可作为"二级审批人"，见下文"审批人操作部分"） |
| 班组长 | 审批处理（可作为"一级审批人"，是考勤组审批人名单里的常见角色） |
| 普通员工 | 打卡、查看本人考勤、提交申请 |

⚠️ 和旧版说明书的差异：以前只区分"管理员/班组长/普通员工"三种，现在**主管和班组长是两个独立角色**，分别对应审批流程里的"二级"和"一级"，详见第四节"考勤组管理"和第三节"审批人操作部分"。

## 二、普通员工操作指南

### 2.1 打卡（菜单：我的打卡）

- 上班点"上班打卡"，下班点"下班打卡"，系统自动记录时间（精确到分钟）。
- **新增"午间打卡"按钮**：部分考勤组的班次配置了"午间必须打一次卡"的时间窗口（比如中午 12:00–13:00），这类班次的员工中午要额外打一次卡，不然下午会被当成"上午没上班"只算下午工时。没配置这个窗口的班次不受影响，正常只打上下班两次卡。
- 打卡后立即显示状态：正常 / 迟到 / 早退，迟到会提示迟到分钟数。
- 若考勤组开启了定位打卡，需允许浏览器定位；超出打卡范围会被拒绝并提示距离。**现在一个考勤组可以配多个打卡地点**（比如总部大楼、分部仓库各算一个），只要落在其中任意一个的范围内都算打卡成功，不用非得在同一个地点。
- 夜班（跨天班次）员工半夜打下班卡，系统会自动把这次下班卡和前一天的上班卡对应上，不会出现"今天没打卡"的误报。
- 节假日 / 公司休息日系统自动标记，无需打卡；在假期日打卡会提示今天无需打卡。

### 2.3 提交申请（菜单：提交申请）

支持 **四类申请**：补卡 / 请假 / 加班 / **出差（新增）**。

选择申请类型并填写信息：
- **补卡**：日期 + 上/下班 + 时间 + 原因；补卡日期不能选未来。
- **请假**：类型 + 起止时间 + 事由；开始时间最早只能选到"现在往前推 24 小时"以内（比如现在是周三下午 3 点，最早能请到周二下午 3 点开始的假），不能选更早的时间。
- **加班**：起止时间 + 事由；**现在只能申请当天的加班**，而且必须在当天 24 点前提交，过了当天就不能再为之前的日子补报加班了。
- **出差（新）**：起止时间 + 目的地（可选）+ 事由；出差不能选过去的时间（出差是提前申请的，不支持事后补报）。审批通过后，出差期间自动按全勤算工时，不用打卡。

⚠️ **"申请原因"现在是必填项**，四类申请都不能留空提交。

⚠️ **审批人现在通常需要手动选择**：如果你所在的考勤组配置了审批人名单，提交申请时会出现"审批人"下拉框，必须从名单里选一个人（一般是你的班组长）；如果没配置名单，系统会自动退给你的直属上级审批，不用你选。

可上传佐证材料（最多 5 个）：逐个添加，列表里每个文件后有"移除"可单独删除，确认无误再提交。

在"我的申请"查看处理状态：待审批 / 审批中 / 已通过 / 已驳回；未处理的可撤销。审批通过后系统自动回写至考勤（补卡补填时间、请假/出差把对应日期标为对应状态、加班累加工时）。

## 三、审批人操作部分（班组长 / 主管 / 文员 / 管理员）

### 处理审批（菜单：待我审批）

- 列表显示待你处理的申请，含申请人、类型、申请时间/日期、事由，**现在列表里直接显示请假/加班的具体起止时间、补卡的具体日期**，不用点进详情才能看到关键信息。
- 列表上方可以按"全部/补卡/请假/加班/出差"分类查看，标签上带每类的数量。
- 点某条申请查看详情，可查看申请人上传的附件。
- 点"通过"或"驳回"，驳回需填写驳回原因；通过不强制填审批意见。
- **新增批量处理**：每行前面有勾选框，可以一次勾多条，点"批量通过"或"批量驳回"一次性处理，批量驳回同样必须填写驳回原因。
- 处理结果会通知申请人。
- "待审批"列表每 30 秒自动刷新；操作弹窗时会自动暂停刷新，不打断操作。

### 关于"二级审批"

⚠️ **系统现在支持审批走一级或二级，具体走几级由管理员在"考勤组管理"里针对每个考勤组配置**（详见第四节 4.3）：

- **一级审批**：只需要你（班组长）审批一次，通过即整单结束。
- **二级审批**：你（班组长）审批通过后，系统会自动把申请再转给申请人自己的"直属上级"（通常是主管角色）审批一次，两级都通过才算最终通过；你这一级驳回的话，直接整单驳回，不会再转给二级。

作为审批人不需要关心自己的申请具体走几级，系统会自动按考勤组的配置流转，"待我审批"里只会显示当前轮到你处理的那一级。

## 四、管理员操作部分

### 4.1 数据看板（菜单：数据看板）

- 首页显示今日实时统计：应到、已打卡、迟到、请假、未打卡、**定位异常（新增）**等，每 30 秒自动刷新数据。
- 点开任意一张统计卡片可以看到该类别的具体人员名单。

> 【配图】数据看板页面截图

### 4.2 员工管理（菜单：员工管理）

- 新增员工：填工号、姓名、角色、部门、考勤组、岗位、手机号、入职日期等并设初始密码。
- **工号可以按部门自动生成**：选好部门后，如果这个部门属于已知的几个公司之一，工号会自动按"公司前缀+流水号"填好，也可以手动改。
- **直属上级现在是必填项**：选好部门后，系统会自动按"该部门下角色是主管的在职员工"把直属上级带出来；如果这个部门还没配主管，或者配了不止一个主管，系统不会自动选，需要手动指定。直属上级会影响审批流程的推送对象（一级审批没配名单时的兜底审批人、二级审批的审批人）。
- 手机号现在新建员工时必填，用于以后"忘记密码"自助找回。
- 入职日期一定要填写，不写员工不能打卡、不能排班（会提示"未办理入职"）。
- 编辑操作：可修改员工信息；"重置密码"会生成新随机密码并显示，也可以手动指定新密码。
- 停用 / 启用：停用后员工立即无法登录（提示"账号已停用"），已登录会话会被强制退出。
- **新增拉黑功能**：拉黑=永不录用，和普通"停用"是两回事，拉黑的工号以后不能再用；可以"移出黑名单"改回普通停用状态。
- **新增批量启用/批量停用**：勾选多个员工后一次性处理。
- **新增员工扫码自助登记**：员工管理页有"待确认登记"标签页，可以生成一个二维码链接，员工自己扫码填姓名/手机号/身份证号/岗位等信息提交，管理员在这里"确认录入"补上工号/部门/考勤组等信息后才正式建号，不用管理员一个字一个字帮员工录入基础信息。
- 支持从钉钉考勤系统网页中映射员工信息、同步钉钉的打卡信息。

- 部门可以设置"长期跟随的考勤组"：设置后，这个部门以后新入职/调入的员工会自动归入对应考勤组，不用每次手动选。

> 【配图】部门管理页面截图

### 4.5 考勤组管理（菜单：考勤组管理）

各分子公司/部门可建立独立考勤组，配置内容包括：

- 午休/晚餐扣除时长。
- 定位打卡：**现在支持配置多个打卡地点**（原来只能配一个），员工落在任意一个地点范围内都算打卡成功；地图选点功能可以直接在地图上点/搜地址，不用手动查经纬度。
- **审批人名单（重要）**：勾选哪些人可以作为本组申请的第一级审批人，候选范围**收窄为"班组长"角色**（以前任何非普通员工都能选）；这是必填项，至少要选一个人。
- **审批层级（新）**：选"一级审批"还是"二级审批"——一级只需要选中的班组长审批；二级在班组长审批通过后，会自动再加一级，交给申请人自己的直属上级（通常是主管）审批，详见第三节的说明。
- 所属部门：勾选哪些部门长期跟随本考勤组。

⚠️ 旧版说明书里提到的独立"审批流程"菜单页已经**不存在**了，审批人和审批层级的配置现在都合并到这个"考勤组管理"页面里，不用再去别的地方单独配置。

### 4.6 班值排班（菜单：班值排班）

- 定义班次：上/下班时间、迟到/早退容忍、是否跨天（夜班）、标识颜色。
- **新增午间必打卡窗口配置**：需要员工中午必须打一次卡的班次，可以配置窗口开始/结束时间（两者要么都填要么都不填）。
- 批量排班：选班次 + 日期区间 + 员工（或勾"适用全组"）→"确认排班"；**现在支持跨多个考勤组合并选人**，不用局限在一个组里挑。
- 页面下方"已排班情况"按日期+班次展示具体排班人员，可按日期范围筛选。
- 已停用员工和未入职员工自动跳过。

> 【配图】班次新增/编辑弹窗截图（含午间必打卡窗口配置）

### 4.7 假期设置（菜单：假期设置）

- 添加法定节假日、公司休息日、调班补班日；标记后对应日期自动免打卡（补班日照常出勤）。
- 按年维护。

### 4.8 报表与导出（菜单：月度报表）

这一块变化比较大，整体重做过：

- **月度汇总表现在直接在网页上就能看全**：以前只有导出的 Excel 才能看到每天的打卡明细，现在网页表格和导出的"模板汇总表"是同一套完整列结构（姓名/部门/工号/每日打卡格子/出勤天数/迟到早退次数/加班时长等），不用下载就能直接看。
- **统计周期可以自选起止日期**，不再局限于自然月（比如可以按"上月26号到本月25号"这种发薪周期统计）。
- 支持按公司/部门范围勾选、按姓名或工号搜索。
- **导出打卡时间表（新按钮）**：在"导出模板汇总表"按钮旁边，导出内容是选中范围内所有人每一天的具体打卡时间、工时、迟到情况等，格式和"导出明细"看到的一样，适合按部门一次性核对大家的具体打卡时间。
- 每人一行仍保留"导出明细"按钮，单独下载这个人当月每日打卡的详细表格。
- 月度汇总每月自动生成，也可手动"生成月度汇总"。

## 五、常见问题解决

| 问题 | 处理 |
|---|---|
| 打卡提示"超出有效范围" | 考勤组开了定位打卡，请到打卡点范围内再打，并允许浏览器定位；一个考勤组可能配了多个打卡点，只要在其中一个范围内就行。 |
| 打卡提示"未办理入职" | 管理员未填入职日期或日期在未来，联系管理员补录。 |
| 登录提示"账号已停用" | 账号被停用，联系管理员启用。 |
| 忘记密码 | 可以自己在登录页点"忘记密码？"用工号+手机号+钉钉验证码自助找回（需要账号已绑定钉钉）；找不回的话联系管理员在"员工管理"里重置密码。 |
| 提交申请时"请选择审批人"报错 | 说明你所在的考勤组配置了审批人名单，必须从下拉框里选一个人，不能空着提交。 |
| 请假开始时间选不了 | 现在请假开始时间最早只能选到"现在往前推 24 小时"以内，选更早的时间会被拦下。 |
| 加班申请选不了之前的日期 | 现在加班只能申请当天的，且要在当天 24 点前提交，不支持补报之前的加班了。 |
| 提交申请没填"申请原因"报错 | 四类申请现在"申请原因"都是必填项，补上就能提交。 |
| 新建员工时"请选择直属上级"报错 | 直属上级现在是必填项，如果部门没自动带出来（该部门没配主管，或配了不止一个），需要手动选一个。 |
| 提交申请要等多久 | 在"我的申请"看实时状态；审批人处理后会收到通知。 |
| 页面数据不是最新 | 看板和待审批每 30 秒自动刷新；其它页面手动刷新浏览器。 |

## 六、注意事项

- 首次登录请尽快修改默认密码；管理员账号尤其要改。
- 新员工入职后务必填写入职日期，否则无法打卡/排班。
- 定位打卡需要在 HTTPS 下允许浏览器定位。
- 提交申请部分附件单个不超过 10MB，每次申请最多提交 5 个文件。
- **申请原因现在是必填项**，四类申请都要填。
- **新建员工时，手机号和直属上级都是必填项**，方便以后审批流程和自助找回密码能顺利跑通。
