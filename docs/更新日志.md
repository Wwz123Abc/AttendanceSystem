# 考勤系统更新日志

> 说明：这份文档记录考勤系统每次功能改动，语言尽量简单，方便非技术人员看懂。
> 每次改动都会在最上面加一条新记录（时间倒序）。
> 同一天的完整 Word 版汇报见本目录下对应日期的 `.docx` 文件。

## 2026-07-08

### 新员工扫码登记页新增"岗位""劳务公司""家庭住址""紧急联系人""身份证照片"
- 以前新员工扫码自助登记，只能填姓名、手机号、身份证号这三项，其它资料都要等管理员当面问了再手动补录。
- 现在扫码登记页多了这几项，员工自己在手机上就能一次性填完：
  - **岗位**：下拉选择，固定七个选项——电气熟手、机械熟手、电气中工、机械中工、普工、调试熟手、调试中工；
  - **劳务公司**（文字填写）；
  - **家庭住址**；
  - **紧急联系人**：姓名 + 电话；
  - **身份证照片**：可以直接拍照上传（jpg/png/webp，最大 10MB），也可以先不传、之后再补。
- 除了岗位是必选，其余几项都是选填，不影响老流程——员工原来只填三项也能正常提交。
- 管理员在"待确认登记"里点"确认录入"时，这几项（包括已上传的身份证照片）会自动带到"新建员工"弹窗里，不用再问员工一遍、也不用重新上传照片；管理员仍然可以在弹窗里改动或补充。

### 新增"员工信息"整合展示页，员工资料补充"紧急联系人""家庭住址""身份证照片"
- 以前想看一个员工的完整资料，得在"员工管理"的编辑弹窗里一项一项看，而且弹窗里也没有身份证照片、紧急联系人这些信息。
- 现在"员工管理"列表每一行新增一个"详情"按钮，点开是一个新的"员工信息"页，把这个员工系统里已知的所有资料——姓名/工号/角色/部门/考勤组/上级、手机号/住址、身份证号/身份证照片、紧急联系人姓名和电话、合同公司/入职日期、钉钉绑定状态/创建时间/最近登录时间——整合在一页展示，方便一次看全，也可以直接点"打印"存档。页面左侧可以按姓名或工号搜索、快速切换到其他员工，不用每次都退回列表重新点。
- 员工资料新增三项内容，在"员工管理"的新建/编辑弹窗里补录：
  - **紧急联系人**：姓名 + 电话（比如员工的家属，出意外时用来联系）；
  - **家庭住址**；
  - **身份证照片**：支持上传 jpg/png/webp 格式图片（最大 10MB），上传新照片会自动替换旧的，不上传则保持原样不变。
- 这三项都是选填，不影响老员工的资料，也不会同步到钉钉通讯录（钉钉那边本来就没有这几项）。
- 这次改动只是新增字段和新增一个查看页面，"员工管理"原有的新建/编辑/停用/删除等操作方式不变。

## 2026-07-07

### 审批流程：考勤组可以配多个审批人，员工提交申请时自己选审批人
- 以前审批人是怎么定的：要么由管理员在"审批流程管理"页给每个考勤组固定配好一串审批人（先审给谁、通过了再审给谁），要么什么都不配就默认走员工的"直属上级"，员工自己没法选。
- 现在改成：管理员在"考勤组管理"页给每个考勤组勾选一个或多个审批人（同一级别，谁都能审）；员工提交请假/补卡/加班/出差申请时，页面上会出现一个"审批人"下拉框，从组里配的这几个人中自己选一个，选完这个人批准/驳回，这条申请就算走完，不会像以前有的组配置多级审批那样还要再转给下一个人审。
- 组里如果一个审批人都没配，规则不变：还是自动交给员工的直属上级，上级也没有的话由系统指派一名管理员/文员兜底，不会出现"没人审"的情况。
- 原来的"审批流程管理"页面已下线（功能被上面这个更简单的方式取代），之前如果给某些考勤组配置过的多级审批流程会失效，需要到"考勤组管理"页重新给对应的组勾选审批人。

### 新增员工可以扫二维码自己填基础信息，管理员确认后再正式建号
- 以前新增员工，所有信息（姓名、手机号等）都要管理员一个字一个字帮忙敲进系统。
- 现在"员工管理"页多了一个"待确认登记"标签页，里面有一个固定的登记二维码（可以打印出来贴在前台，或者把链接发给新员工）。新员工用手机扫码打开一个网页，自己填姓名、手机号、身份证号提交，不需要登录、也不需要管理员在场。
- 员工提交后，会出现在管理员的"待确认登记"列表里。管理员点"确认录入"，会弹出和"新建员工"一样的窗口，姓名/手机号/身份证号已经自动填好，管理员只需要再补上工号、部门、考勤组、角色等信息，保存后就正式建好账号了；如果这条登记有问题（比如填错了、不是真的要入职），点"驳回"即可，不会建账号。
- 同一个手机号或身份证号如果已经是系统里的员工、或者已经提交过还在等待确认，再扫码提交会被挡下并提示，不会重复产生登记记录。
- 员工表单里新增了"身份证号"这一栏（选填），编辑老员工信息时也可以补录。

### 彻底删除员工时，联动删除对应的钉钉账号
- 以前系统里"彻底删除"一个绑定过钉钉的员工，钉钉通讯录里那个人还留着，得手动去钉钉后台再删一遍。
- 现在点"彻底删除"，如果这个员工绑定了钉钉，系统会同时调用钉钉接口把他从企业通讯录里删掉，两边保持一致。
- 说明：钉钉只提供"删除"，没有"临时禁用"这种操作，所以这个联动**只放在"彻底删除"这个本来就不可恢复的操作上**，"停用""拉黑"这两个可以恢复的操作不会去动钉钉那边的账号。如果钉钉那边删除失败（比如网络问题），本地删除仍然会正常完成，页面会提示"请到钉钉通讯录手动确认"。

### 员工信息新增"合同公司"
- 员工管理页的新建/编辑表单里新增"合同公司"一栏（选填），用来记录员工实际签合同、发工资的公司主体（有的员工所在部门和签合同的公司主体不是同一家，这里可以分开记录）。
- 员工列表也新增了"合同公司"这一列，方便一眼看到。

### 修复：编辑员工时"身份证号"填了不生效
- 上一条新增"身份证号"字段时遗漏了保存逻辑，导致编辑老员工、修改身份证号后点保存不会真正存进去（新建员工时是正常的，只有"编辑"这条路径受影响）。这次一并修复。

### 考勤组：支持配多个打卡地点，并对钉钉打卡做定位比对；取消"一个部门一个考勤组"
- **多个打卡地点**：以前一个考勤组只能配一个打卡点（一个经纬度+一个半径）。现在可以给一个考勤组配好几个打卡地点（比如"总部大楼"+"分部仓库"），员工落在其中任意一个地点的有效范围内，打卡就算数，不要求同时满足所有地点。
- **钉钉打卡定位比对**：以前从钉钉同步过来的打卡记录，只是原样存进系统，从来没有核对过打卡地点靠不靠谱。现在同步时会拿钉钉上报的定位，和员工所在考勤组配置的打卡地点比对：如果这天有任何一次打卡的定位和所有地点都对不上（超出有效范围），这一天会被标记"定位异常"，出现在"数据看板"首页的统计卡片里，员工自己在"我的记录"页也能看到一个小图标提示。**这个标记只是提醒管理员去核实，不会自动影响当天的迟到/旷工等考勤判定结果。**
- **考勤组的"所属公司"改成勾选部门**：以前"所属公司"是手动打字填的一个文本框，和真实的部门数据没有关联；另外系统里有"一个部门自动对应一个考勤组"的设定（新建部门会自动建一个同名考勤组，员工换部门也会被强制换组）。
  现在这两点都改了：新建/编辑考勤组时，"所属公司"变成一个部门树状勾选框，勾选哪些部门就表示这些部门"长期跟随"本考勤组——这些部门现有的员工会立刻批量归入本组，以后新入职/调入这些部门的员工也会自动归入，不用每次手动选；"所属公司"文字则根据勾选的部门自动带出来，不用再手动打字。同一个考勤组现在可以对应多个部门（不再是"一个部门一个组"）。
  **需要你知道**：以前如果用过"按部门同步"功能，那些自动建的对应关系已经保留（迁移时已把数据搬到新结构里），但"按部门同步"这个按钮本身已经下线，以后新的部门/考勤组对应关系需要到"考勤组管理"页手动勾选部门来配置。

### 员工列表：操作按钮排成一行
- 以前"编辑/停用/拉黑/重置密码/删除"这几个按钮挤在一起会自动换成两行，看着比较乱。现在统一排成一行、按钮之间留出统一间距，屏幕较窄时表格会整体左右滚动，不会再把按钮挤成两行。

### 修复：员工管理"待确认登记"标签页二维码不显示、按钮点了没反应
- 这块页面代码里的引号不小心被写成了中文弯引号，导致浏览器认不出样式和按钮点击事件，看起来整块内容都是"裸奔"的（没有卡片边框、按钮样式，二维码也加载不出来）。这次改回正常引号并顺带整理了一下这块的样式（加了图标、卡片间距更统一）。

### 考勤组管理：勾选部门树时，勾大部门会连带勾上它的所有下级部门
- 以前在"新建/编辑考勤组"里勾部门树时，每个部门都要一个一个单独勾，勾了"科瑞科技"这个大部门，下面的"BD1""BD2"等小部门不会跟着选中。现在勾选一个部门会自动连带勾上它所有的下级部门（取消勾选也会连带取消下级），不用再逐个手动勾。

### 考勤组管理：新增"删除"功能
- 以前考勤组只能"停用"，没法彻底删掉。现在列表里每行加了一个删除按钮，点击会先弹窗说清楚删除的影响（组里员工的考勤组归属会被清空、跟随本组的部门会解除跟随、组下的班次/排班记录会一并删除），确认后才会真正删除，且不可恢复。

### 班值排班：新增班次时可以自由选择"每周休息日"
- 以前系统里"周末休息"是写死的规则（固定周六、周日），没法给三班倒、轮班这类特殊班次单独设置不一样的休息日。
- 现在"新增/编辑班次"弹窗里加了一排"周日~周六"的勾选框，勾哪几天，这个班次的人这几天就不用上班——比如可以设成"休二、三"这种非标准的休息模式，不再局限于周六周日。
- 这个设置会影响"旷工"判定：员工当天如果被排了这个班次、又刚好是这个班次配置的休息日，系统就不会因为没打卡把他标记成旷工（公司统一安排的"调班补班日"优先级更高，就算撞上班次自己配的休息日，那天也还是要正常上班）。
- 老班次不受影响：升级后已有的班次会自动按"周六、周日休息"补上，行为和以前一样，需要的话可以再进"编辑"改成别的组合。
- 已知限制：月度报表里的"应出勤天数"目前仍按"周末+法定节假日"统一计算，还没有跟着每个班次自己的休息日走，如果班次的休息日跟标准周末不一样，报表里的应出勤天数可能会有些许出入，后续可以再优化。

### 月度报表：新增"模板格式月度汇总表"导出，按公司要求的表格样式来，可按部门/公司筛选
- 公司要求月度汇总表按固定的外部模板格式来（比如日期用"上月26号至本月25号"的结算周期、每人一行、每天一格打卡工时、外加一堆迟到/早退/缺卡/加班等细分统计），和系统原来的"导出汇总表"格式不一样，所以新增了一个独立的导出入口，原来的"导出汇总表"/"导出个人明细"不受影响，照常能用。
- 月度报表页新加一块"模板格式月度汇总表"，选好年份/月份后（页面上方那组"年份/月份"），点"导出模板汇总表"就能下载，统计周期自动换算成"上月26号至本月25号"（比如选"2026年6月"，导出的就是"2026-05-26 至 2026-06-25"这段时间的数据，和公司模板的周期规则对齐）。
- 导出范围用一棵"公司/部门"合并的勾选树来选，公司和部门合在一起、范围可大可小：勾选一个公司节点，会自动连带勾上它下面所有部门（一次导出整个公司）；只勾某个具体部门，就只导出这个部门（范围缩小到一个小组）；两种可以混着勾，也可以什么都不勾（默认导出全公司所有人）。
- 每天的打卡工时按要求做成整数（比如 7.5 小时会显示成 7，不四舍五入），但月度合计的"工作时长"这一列还是保留小数，方便对总数。
- 有几点和公司模板略有差异，属于系统目前能力范围内的合理简化，实际使用中如果发现对不上可以再反馈调整：
  - "出勤天数"按整天计算（有没有工时算 1 天或 0 天），没有做半天/半日这种精细颗粒度的区分；
  - "加班总时长/工作日加班/休息日加班/节假日加班"这四列，是系统按每天实际的日期类型（工作日/周末或班次自己的休息日/法定节假日）分别精确算出来的，不是简单地把同一个数字复制三遍；
  - "外出时长"这一列系统里目前没有对应的考勤类型，会一直是空白。

### 数据看板：去掉"快速导航"卡片
- 数据看板首页原来在"出勤率"旁边有一块"快速导航"（员工管理/月度报表/假期设置/待审批的快捷入口），现在按要求去掉了，"出勤率"卡片改成占满整行。

### 修复：月度报表"导出模板汇总表"点了没反应
- 原因：这个按钮是表单提交（不是普通链接），提交方式又是最常见的"GET"，浏览器对 GET 方式的表单有个特殊规则——不管地址栏原来写的是什么，提交时只会用表单里的字段重新拼一遍网址，所以原来藏在地址里的"要走哪个导出功能"这个标记会被浏览器悄悄丢掉，请求就退回成打开这个页面本身，看起来就像点了没反应。
- 修复方法：把"要走哪个导出功能"这个标记也放进表单的一个隐藏字段里，跟着年份、部门一起提交，这样就不会被浏览器丢掉了。
- 顺带去掉了页面顶部旧的"导出汇总表"按钮（格式和现在的"模板格式月度汇总表"重复，已经用不上）。

### 员工管理："新建/编辑员工"弹窗里的"部门"下拉改成树状显示
- 以前这个下拉框里的部门是打乱的一份平铺列表，看不出哪个部门是哪个部门的下级。现在改成按公司部门架构的树状顺序排列，子部门名字前面会有缩进，一眼就能看出上下级关系（和"部门管理"页选择"上级部门"时的显示方式一致）。
- 检查了系统里其他选部门的地方："部门管理"页选"上级部门"那个下拉本来就是树状缩进显示，不用改；"考勤组管理"和"月度报表"里的部门选择是勾选框形式的树，本来就有树状结构，也不用改。

### 编辑员工信息时，同步更新到钉钉通讯录
- 以前系统里改员工的姓名、手机号、工号、职位，只会改本地数据库，钉钉通讯录那边的资料不会跟着变，两边容易对不上。现在编辑一个**已经绑定过钉钉**（有钉钉账号）的员工时，保存的同时会把这几项同步更新到钉钉通讯录对应的那个人身上。
- 如果钉钉那边同步失败（比如网络问题、手机号在钉钉企业内被别人占用了），本地信息仍会正常保存，只是页面上会多提示一句"请到钉钉通讯录手动确认"，不会因为钉钉那边的问题导致本地保存失败。

### 新建员工时，也能同步创建到钉钉通讯录
- 新增一名员工时，如果**同时满足两个条件**——填了手机号、所在部门是从"从钉钉导入通讯录"关联过来的（能换算出对应的钉钉部门）——本地保存成功后，会顺带在钉钉通讯录里也建一个新账号，并把钉钉那边分配的账号编号记下来，以后编辑/删除这个人都会继续跟钉钉联动。
- 两个条件缺一个（没填手机号，或者部门是本地手工建的、跟钉钉对不上）就直接跳过，不会强行创建，也不提示报错——很多临时工本来就不需要开钉钉账号，这是正常情况，不是失败。
- 提醒一下：钉钉建账号这个动作**不可逆**（钉钉那边只支持"删除"，不支持"撤销新建"），如果手机号在钉钉企业内已经被别人用过，会直接创建失败；失败不会影响本地新建员工，只是会多提示一句"请到钉钉通讯录手动添加"。

### 钉钉同步失败时，提示里直接带上失败原因
- 以前新建/编辑/删除员工遇到钉钉同步失败，页面上只会显示一句很笼统的"钉钉同步失败，请到钉钉通讯录手动确认"，具体是什么原因（比如手机号被占用、部门对不上）看不出来，得去服务器的日志文件里翻，对不熟悉技术的人来说基本没法用。
- 现在这几句提示会直接把钉钉那边返回的具体错误原因显示出来，不用再去找日志文件。

### 修复：编辑员工同步钉钉时，"手机号跟员工本人登录钉钉的手机号不一致"会导致姓名/工号/职位也一起同步不上
- 发现的问题：钉钉有一条官方限制——如果员工本人是用另一个手机号登录钉钉的（跟系统里存的手机号不一样），钉钉会直接拒绝这次"更新用户"请求，报错"企业中的手机号码和登陆钉钉的手机号码不一致"。之前系统是把姓名、手机号、工号、职位打包一起发给钉钉，只要手机号这一项被拒绝，**整个请求都会失败**，姓名、工号、职位这些明明没问题的信息也跟着同步不上。
- 修复方法：遇到这种"手机号不一致"的情况，系统会自动去掉手机号这一项再发一次，让姓名、工号、职位照常同步过去，只有手机号这一项同步不了。页面上的提示也会更准确地说明是"手机号没同步，其余资料已同步成功"，不再是一句笼统的"同步失败"。
- 这条限制是钉钉那边定的，不是系统的问题——钉钉官方的说法是"可以删除后重新添加"才能改这个手机号，本系统目前没有做这一步（删除重建的风险比较大），如果确实需要改，只能到钉钉通讯录里让员工本人自己改，或者线下走钉钉的账号处理流程。

### 修复：月度报表导出的"模板格式月度汇总表"里，加班相关的几列数字算错了
- 发现的问题：Excel 里"加班总时长""工作日加班""休息日加班""节假日加班"这四列，单位本来应该跟"工作时长"一样是"小时"，但代码里被误当成了"分钟"来处理，多乘了一次 60，导致导出的数字比实际加班时长大了 60 倍（比如实际加班 110 小时，导出的表格里会显示成 6600 多）。这次已经改正，这四列现在跟"工作时长"一样是小时数，不会再乘错。
- 顺带发现并修复了一个相关问题：这份导出报表里，如果员工是月中才入职的，入职之前那几天会被误判成"旷工"（因为那几天本来就没有考勤记录，系统之前没有排除"还没入职"这种情况）。现在会正确跳过入职日之前的日子，不再把新员工没入职前的空白算作旷工。

### 全面审查月度报表相关代码，修复三处"员工离职后数据凭空消失"的问题
- 按要求把月度报表牵扯到的代码全部重新审了一遍，发现并修复了三处同一类问题——**员工只要被停用（离职），他那个月哪怕确实上过班，也会从报表里直接消失或者算不出来**，具体是：
  1. **月度汇总生成**：以前系统只给"当前还在职"的员工算月度汇总，如果一个人这个月工作到一半才离职，系统还没来得及给他算这个月的汇总时他就已经被停用了，之后就永远不会给他补算了，等于这个月的记录凭空消失。现在改成：只要这个人现在在职，或者这个月确实有打卡记录，或者以前已经给他算过这个月的汇总，都会正常处理，不会因为后来离职就漏掉。
  2. **月度报表主表**（页面上那张"部门考勤汇总"表）：以前查询时只显示"当前在职"的员工，哪怕某个离职员工那个月的汇总数据其实已经算好、存在数据库里，也会因为人已经不在职而被过滤掉、看不到。现在改成不按"是否在职"过滤，只要那个月有数据就能查到。
  3. **模板格式月度汇总表（发工资用的那份导出）**：这个问题影响最大——以前导出时也是只导"当前在职"的员工，如果有人在这个薪资结算周期（比如"上月26号至本月25号"）里离职，导出的时候人已经被停用了，就会**整个人从表格里消失**，那个月的工资就没法从这份表算出来。现在改成把"这段周期里有考勤记录的人"也一并纳入，不会再漏人。
- 这几个问题不是这次新引入的，是系统里已经存在一段时间的逻辑漏洞，这次借着"重新审查"的机会一并查出来修掉了。
- 遗留的小限制（不是本次范围）：员工没有"离职日期"这个字段，只有"入职日期"，所以离职员工最后那个月的"应出勤天数"暂时还是按整月算的，没有在离职当天截止，可能会比实际略多算几天应出勤（不会导致人从报表里消失，只是这一个数字可能有些微出入）。如果需要更精确，后续可以加一个"离职日期"字段来解决，这个改动会涉及数据库结构，先记录在这里，看是否需要再单独处理。

### 修复：模板格式月度汇总表里，还没到的"未来几天"会被当成旷工
- 发现的问题：这份报表的统计周期是"上月26号至本月25号"，只要在25号之前导出（这是最常见的用法，比如月中查一下进度），周期里就会包含"还没到的日子"。这些日子当然还没有考勤记录，但代码把"没有记录"一律当成"旷工"处理，没有区分"这天已经过去但没打卡"和"这天压根还没到"，导致每个人的旷工天数都会被莫名其妙地多算出一大串（比如月中导出，后面十几天全部被当成旷工）。这是这次审查里发现的影响最大的一个问题，几乎每次月中导出都会碰到。
- 修复方法：只有"已经过去、但没有记录"的日子才算旷工；还没到的日子统一显示空白，不再计入旷工。

### 说明：这次审查没有做数据库层面的实测
- 按以往的约定，代码改完不用系统自带的数据库做实际操作来自测，验证交给你来做。这次为了彻底核对报表数字，本来想插几条测试用的员工/考勤数据，实际导出一遍核对，但这个操作涉及往真实数据库里写数据，已经主动放弃，改为逐行重新审查代码逻辑（把上面几处问题揪出来的方式）。所以这次的把握程度是"代码逻辑审查过关"，不是"拿真实数据跑过一遍"，建议你实际导出几个月份、尤其是有人离职/入职的月份，核对一下数字，如果还有对不上的地方随时告诉我。

系统里的临时工都是按工时结算工资的，所以把打卡→工时这条链路完整核查了一遍，补上了几处会导致工时算错/算漏的地方：
- **补卡审批**：以前补卡审批通过后只补上打卡时间，没有重新计算这一天的实际工时和加班时长；现在补卡通过后会自动把工时和加班都重新算好，并且不再把这天错误地记成"旷工"。
- **月度报表**：以前遇到工时是 0 的老记录（早期从钉钉同步、还没来得及算工时的数据），只是在算"月度合计"的时候临时补一下，日报表和导出的 Excel 明细里那一天还是显示 0，跟月度合计对不上；现在会把补算出来的工时直接写回那一天的记录，日明细和月合计从此永远一致。
- **效果**：只要打开一次某个月的"月度报表"页面，那个月所有老数据都会自动被修正（不需要额外操作）。建议对过去几个月都打开一次报表页面，把历史数据都补一遍。

### 修复接口层的两个隐藏问题（不影响日常在网页上操作，主要影响以后对接手机端/第三方系统时的稳定性）
- **问题一：没登录/没权限时，接口不会正确报错。** 以前如果用接口方式（而不是网页表单）调用系统且没登录，系统会返回一个"跳转到登录页"的指令，而不是标准的"未登录"提示，导致对接程序无法正确判断到底是"没登录"还是"没权限"。现在已改为返回标准的"未登录"/"无权限"提示，方便以后对接手机端或其他系统时能正确处理这两种情况。
- **问题二：部分接口出错时，只显示"服务器内部错误"，看不出具体原因。** 比如"新增员工时工号重复""用已停用的账号登录"这类情况，走网页表单操作时都会给出准确的提示（如"工号已存在"），但如果走接口调用，之前会被笼统地当成系统故障处理，看不到具体是哪里错了。现在接口调用也能像网页一样，返回准确的错误原因。
