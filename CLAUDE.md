# 项目说明

考勤管理系统：ASP.NET Core 8 + Razor Pages + EF Core + MariaDB。

## 更新日志维护约定

**每次对本项目完成一项功能性改动（新增/修改/修复）后，需要在 `docs/更新日志.md` 文件的最上方（`---` 分隔线下方）追加一条新记录。**

格式要求：
- 按日期分节（`## YYYY-MM-DD`），同一天的多次改动写在同一个日期节里，追加在该节末尾。
- 语言要简单易懂，避免技术术语堆砌，参考文件里已有条目的写法（"以前怎样 → 现在怎样 → 有什么好处"）。
- 纯内部代码重构、变量改名、注释调整这类不影响功能/界面的改动不需要记录；只记录用户能感知到的变化。
- **每次往 `docs/更新日志.md` 追加完记录后，必须接着跑一遍 `tools/update_changelog_docx.ps1`，
  把 `docs/考勤系统更新日志_完整版.docx` 同步更新（不用问，直接做）：**
  `powershell -ExecutionPolicy Bypass -File tools/update_changelog_docx.ps1`
  这个脚本会用当前 `docs/更新日志.md` 的全部内容重新生成整份 Word 文档（不是增量追加），
  所以每次都是"重新生成"，不用担心漏更新或和 md 内容对不上。

这份日志是给项目负责人看的，不是给开发者看的技术文档 —— 写作时始终假设读者不熟悉代码。
